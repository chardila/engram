# engram — Plugin Instructions

This plugin provides skills for managing a personal Obsidian vault via Claude.

## Setup

Each vault instance has its own `CLAUDE.md` at the vault root with vault-specific instructions.
Read that file first at the start of every session — it defines the vault's structure, MCP server
name, and behavioral protocols.

## Available Skills

| Command | Purpose |
|---------|---------|
| `/daily-prep` | Start the day: read STATE.md, create daily note, draft priorities |
| `/end-day` | Close the day: capture learning, update AI/sessions/, regenerate STATE.md |
| `/process-inbox` | Convert inbox loops into structured notes and tasks |
| `/review-week` | Generate weekly journal, update STATE.md with weekly view |
| `/analyze-vault` | Monthly topological audit: orphans, stale areas, weak signals |
| `/review-system` | Monthly meta-review: usage patterns, friction, system improvements |

## Key Conventions

- All vault content in Spanish
- Filenames in kebab-case
- Frontmatter YAML in every note (created, tags, status, related, author)
- `author: user` for user-written content, `author: claude` for Claude-generated content
- Tasks use Obsidian Tasks plugin syntax: `- [ ] task 📅 YYYY-MM-DD #tag`
- Wiki-links `[[note-name]]` for concept connections
- STATE.md is auto-generated by Claude — `## Hechos duros` block is manual-only

## Session Start Protocol

1. Identify which vault is active (brain-work or brain-personal)
2. Read that vault's `CLAUDE.md`
3. Execute `/daily-prep` or pick up from last session log in `AI/sessions/`
